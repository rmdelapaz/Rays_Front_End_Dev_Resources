JavaScript Primitives

What are Javascript primitives

Primitive is the designation used for different types of indivisible units of data. It could be someone's first name, age or if it's raining outside (true or false). Yes, you can break down somesone's first name into it's individual components but those individual components by themselves are irrelevant unless that is what you need. (the first later of the first name might be needed as the initial)

Types of primitives

Strings

For non programmers, strings can be for guitars, kites and quantum theory. I've had a student mention string cheese. In javascript and most programming languages (check to be sure), any characters or text contained in single or double quotes will be considered a string. It is called string because it is a sequence of letters orignially referred to as a "string of characters".

Numbers

The amount or how much of something (quantity/price) is usually expressed in numerical fashion. Numbers are primitives we could used math on. Using math on strings is silly.

Boolean

Named after an english mathematician, George Boole who came up with algebraic notiation used to represent logical propositions. A boolean is either true or false. A boolean expression is something that determines or results in a boolean (true or false). We learn more about this with conditionals aka if statements.
