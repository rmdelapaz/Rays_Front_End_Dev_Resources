HTML Cheat Sheet

Overview

This section can just serve as introductory paragraph samples but does contain some useful information. Also, worth mentioning that this is a single simple html file without any CSS or Javascript. A web page without any decoration.

This is a sample of various html tags in action. If you are new to web development, and viewing this in a browser, you are seeing the result of the html, try viewing the source along side to reveal the HTML code that the browser is rendering

HTML stands for Hyper Text Markup Language, not a programming language but a simple markup language that "marks up" or "tags" content to identify it for the browser. The browser will then know how to treat/display that "element". That is how HTML calls it fully marked up item. An element. Tags are HTML markup around content or information. The whole unit together is an element.

Typically tags come in pairs, an opening tag that delineates the start of the content and an closing tag to define where it ends. Tags are comprised of less than symbol '<' followed by the tag name followed by a greater than '>'. For example <p> would be an opening p tag. For those examining this in html code instead of the browser view, note that I had ot include the necessary html markup to create a less than and a greater than for display rather than having the display be handled as actual paragraph markup. Majority of html tags require a closing tag. Closing tags have a / preceding the tag name (observe closing paragraph tags).

Text emphasis with em and strong

Before we discuss em and strong, worth mentioning how tags have built in display behaviors. These default display behaviors can be modified. The heading tags you saw in the beginning have the default display behavior of block style. That means they don't like to share the display line with anything else. If placed within content, it will start on a new line and have following text start on the next line. This is display block. The other most common disply behavior is inline which as it's name implies, stays inline. An example of inline we've seen so far is the anchor tag. You can have anchor tags around an individual word or letter inside of other content and the display of it will be inline along the other content.

Sample of emphasis and strong

The em tag is short for emphasis text with em tag which displays inline in italic text while when we use the strong tag, what it does is strong tag applied makes the text appear bold. Modern web development prefers the term emphasis and strong for accessibility reasons. Old ways had i and b tags which may still function but should be avoided

Tags to further organize your content

div and span tag samples

There are tags to such as div and span that you can use to group similar elements and separate different elements as necessary. Worth noting, div is a block style and span is inline. Use accordingly. Typically, div is used for larger pieces of content while span would be for smaller portions.

Further uses of the div tag in conjuntion with other attributes allow you to identify uniquely or classify into a group various elements. More on that later, especially when we get into CSS and JavaScript

Hopefully by now, creating and using tags to organize your content is easy, putting what you need in between appropriate tags. The challenge will be when do I need this tag over that tag.
